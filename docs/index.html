Core concepts

Soleri replaces stateless chat loops with durable assistant infrastructure. Knowledge and behavior persist and improve with use.

Vault

Persistent knowledge store for patterns, workflows, and architecture decisions.

Brain

Learning loop that captures intelligence from real sessions and compounds it.

Memory

Cross-session and cross-project continuity without repeated setup or re-teaching.

Personas

Specialized identities with distinct expertise on top of a shared core engine.

Forge

CLI lifecycle for creating, updating, and managing personas in production.

Transports

MCP today, with REST and LSP paths designed into the architecture.

Architecture

Soleri is structured as layered infrastructure: persona-level specialization above a reusable engine and pluggable transports.

Personas

Salvador, Gaudi, Sentinel, and your own custom personas

Domains

Design, security, architecture, and additional pluggable expertise

Engine

Vault, brain, planning, and memory as the shared core

Transports

MCP for Claude Code today, REST and LSP ready tomorrow

Official personas

Salvador

Design systems: tokens, components, accessibility, and visual validation.

Gaudi

Architecture: system design, API patterns, persistence, and performance.

Sentinel

Security: vulnerability analysis, API hardening, and threat modeling.

Start with one persona. Grow into a full assistant system.

Soleri is named after Paolo Soleri and his idea that systems should be living and adaptive. Your assistants should evolve the same way.
