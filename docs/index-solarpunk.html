Layered like living systems

Each layer grows independently. Swap transports, plug in knowledge, create personas — the engine stays the same.

Personas

Thin config layers (persona.yaml): identity, voice, domain wiring, intent rules. Forge generates them, the engine runs them. Switch at runtime — no restart.

Domains

Pluggable npm modules: @soleri/domain-design, @soleri/domain-security, @soleri/domain-architecture. Engine loads/unloads them based on active persona.

Engine

Single MCP server process. Vault (knowledge), Brain (learning), Memory (context), Planning (workflows). All personas share it. Updates via npm update — every persona benefits immediately.

Transports

Protocol adapters isolated from core: mcp.ts ships now, rest.ts and lsp.ts ready for VS Code, Cursor, Zed, and custom dashboards. Adding a transport = one adapter file.
