# v6.5.0 — Scaling & Performance Design

## Goal

Make Soleri production-ready at scale: PostgreSQL persistence provider for concurrent team writes (#85), and vault optimization at 10K+ entries (#86).

## Architecture

Approach A (Provider-First): migrate all 5 raw-SQLite modules to `PersistenceProvider`, build the PostgreSQL provider, then optimize SQLite. This ensures PG is functional end-to-end from day one.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| PG use case | Team/server deployments | SQLite handles scale fine with indexes; PG's value is concurrent writes |
| Module migration | In this milestone | Without it, PG provider is dead code for 5/7 modules |
| PG FTS | tsvector/tsquery + GIN | Built-in, closest to FTS5 porter stemming, zero extra deps |
| PG library | pg (node-postgres) | Battle-tested, built-in Pool, optional dependency |

## 1. PersistenceProvider Interface Extension

Current interface: `execSql`, `run`, `get`, `all`, `transaction`, `close`.

New additions:

- `ftsSearch<T>(table, query, options?)` — Full-text search abstraction. SQLite uses FTS5, PG uses tsvector/tsquery.
- `ftsRebuild(table)` — Rebuild FTS index (FTS5 rebuild or tsvector reindex).
- `readonly backend: 'sqlite' | 'postgres'` — Backend type for conditional schema logic.
- `FtsSearchOptions` — columns, limit, offset, filters, orderByRank.

`ftsSearch` lets modules search without knowing FTS5 vs tsvector. SQLite implementation delegates to existing FTS5 queries. PG implementation uses `to_tsquery()` + `ts_rank()`.

## 2. PostgresPersistenceProvider

- `pg.Pool` with configurable size (default 10 connections), 30s idle timeout.
- **Parameter translation** — SQLite `?` and `@name` params transparently converted to PG `$1, $2` positional style.
- **Schema adaptation** — `SERIAL` instead of `AUTOINCREMENT`, `TIMESTAMPTZ` instead of `INTEGER` for timestamps, `tsvector` columns + GIN indexes instead of FTS5 virtual tables.
- **Optional dependency** — `pg` in `optionalDependencies`, dynamically imported at runtime.
- **Transactions** — `pool.connect()` + `BEGIN`/`COMMIT`/`ROLLBACK` with automatic release.

## 3. Module Migration

5 modules currently use `vault.getDb()` (raw better-sqlite3). Each migrates to `vault.getProvider()`:

| Module | Tables Owned |
|--------|-------------|
| Curator | curator_entry_state, curator_tag_canonical, curator_tag_alias, curator_changelog, curator_contradictions |
| BrainIntelligence | brain_strengths, brain_sessions, brain_proposals, brain_global_registry, brain_domain_profiles |
| Governance | vault_policies, vault_proposals, vault_policy_evaluations, vault_policy_changes |
| ProjectRegistry | registered_projects, project_rules, project_links |
| IdentityManager + IntentRouter | agent_identity, agent_identity_versions, agent_guidelines, agent_modes, agent_routing_log |

Migration pattern: replace `db.prepare(sql).run(...)` with `provider.run(sql, [...])` and `db.exec(sql)` with `provider.execSql(sql)`.

After migration, `vault.getDb()` is deprecated (kept with console warning, removed in next major).

## 4. SQLite Optimization

### Missing Indexes

- `idx_entries_domain` on entries(domain)
- `idx_entries_type` on entries(type)
- `idx_entries_severity` on entries(severity)
- `idx_memories_project` on memories(project_path)
- `idx_memories_type` on memories(type)
- `idx_curator_state_status` on curator_entry_state(status)
- `idx_curator_changelog_entry` on curator_changelog(entry_id)

### Performance Pragmas

- `cache_size = -64000` (64MB page cache, default ~2MB)
- `temp_store = MEMORY` (temp tables in RAM)
- `mmap_size = 268435456` (256MB mmap for reads)
- `synchronous = NORMAL` (safe with WAL mode)

### Entry Archival

New `entries_archive` table mirrors `entries` schema + `archived_at` timestamp.

- `vault.archive(options)` — move entries older than N days with low confidence
- `vault.restore(id)` — bring archived entries back to active table
- Keeps FTS index lean at scale

## 5. Configuration

Environment variables:

- `SOLERI_PERSISTENCE=postgres` (default: `sqlite`)
- `SOLERI_DATABASE_URL=postgresql://user:pass@host:5432/dbname`

Runtime factory auto-detects: if env vars set, use PG; otherwise SQLite file.
