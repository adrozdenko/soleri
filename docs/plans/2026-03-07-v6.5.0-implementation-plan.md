### Task 3: Create PostgresPersistenceProvider

**Files:**

- Create: `packages/core/src/persistence/postgres-provider.ts`
- Modify: `packages/core/src/persistence/index.ts`
- Modify: `packages/core/package.json`
- Test: `packages/core/src/__tests__/postgres-provider.test.ts`

Add `pg: ^8.13.0` to `optionalDependencies`, `@types/pg: ^8.11.0` to `devDependencies`. Run `npm install`.

Create `PostgresPersistenceProvider` class implementing `PersistenceProvider`. Key piece: `translateSql()` function that converts SQLite `?` to PG `$1,$2`, converts `@name` named params to `$N` positional, and replaces `unixepoch()` with `EXTRACT(EPOCH FROM NOW())::integer`.

The PG provider is a **working stub** — `translateSql()` is fully tested and reusable. Full async PG support requires making the `PersistenceProvider` interface async (v7.0 scope). For now, `PostgresPersistenceProvider.create()` is the async factory, and methods document the sync/async mismatch.

`ftsSearch` generates `tsvector/tsquery` queries with `ts_rank()` and GIN index usage.

Export from `persistence/index.ts`.

Tests: `translateSql ? to $N`, `translateSql @name to $N`, `translateSql unixepoch`, `backend returns postgres`.

Run: `cd packages/core && npx vitest run src/__tests__/postgres-provider.test.ts`
