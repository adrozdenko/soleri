# Content Workflow

## Pipeline: Script to Publish

Every piece moves through these stages. No skipping.

### 1. Ideation (scripts/)

- Capture raw ideas as single-sentence hooks in `scripts/ideas.md`
- Expand promising ideas into outlines with: audience, angle, format, goal
- Outline structure: Hook → Problem → Insight → Proof → CTA

### 2. Drafting (production/)

- Write the first draft fast. Do not edit while drafting.
- Name files: `YYYY-MM-DD-slug.md` (e.g., `2024-03-15-why-hooks-matter.md`)
- Include frontmatter: title, format, audience, status (draft/review/final)
- Target word count before you start. Overwriting means under-thinking.

### 3. Editing (production/)

- First pass: structure. Does each section earn its place?
- Second pass: clarity. Can every sentence be shorter?
- Third pass: voice. Does it sound like the author, not a committee?
- Check: hook strength, CTA clarity, formatting for scan-readers

### 4. Publishing (distribution/)

- Adapt the piece per platform (blog, social, newsletter, video)
- Track where each piece was published and when
- Log performance notes: what resonated, what fell flat

## Vault Integration

- After publishing, capture patterns to vault: what hooks worked, what formats performed, audience reactions
- Before starting a new piece, search vault for: topic patterns, style notes, past performance on similar content
- Anti-patterns go to vault too: headlines that flopped, formats that underperformed

## Batch vs. Single

- Ideation and outlining batch well — do 5-10 at once
- Drafting is deep work — one piece at a time
- Editing can batch if pieces are in the same format
- Publishing and distribution batch by platform
