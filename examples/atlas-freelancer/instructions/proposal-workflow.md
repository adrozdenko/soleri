# Proposal Workflow

## Pipeline: Discovery to Delivery

### 1. Discovery

- Ask before assuming. A 15-minute discovery call saves 15 hours of rework.
- Capture: problem, desired outcome, timeline, budget range, decision makers
- Use `templates/discovery-questions.md` as a starting checklist
- Document everything in the client's `brief.md`

### 2. Scoping

- Break the project into phases. Each phase has: deliverables, timeline, dependencies.
- Define what's IN scope and what's OUT scope explicitly — ambiguity is where scope creep lives.
- Identify risks and assumptions. Write them down.
- Create `scope.md` in the client folder with all of the above.

### 3. Pricing

- Price the value, not the hours.
- Three-tier options when possible: Basic / Standard / Premium
- Include revision limits, timeline, and payment schedule
- Never discount without removing scope. Price integrity matters.

| Pricing Model | When to Use |
|--------------|-------------|
| Fixed price | Clear scope, defined deliverables |
| Retainer | Ongoing relationship, variable work |
| Hourly | Exploratory work, consulting |
| Value-based | High-impact projects with measurable ROI |

### 4. Proposal

- Use `templates/proposal.md` as the base structure
- Structure: Executive Summary → Problem → Solution → Scope → Timeline → Investment → Next Steps
- Keep it under 3 pages. Executives skim; respect their time.
- End with a clear next step and deadline for response.

### 5. Delivery

- Milestone check-ins at each phase boundary
- Deliverables go in `clients/<name>/deliverables/`
- Get sign-off before moving to next phase
- Invoice on delivery, not on completion of all phases

## After Project Closes

- Capture lessons to vault: what went well, what you'd change
- Update templates if you found a better approach
- Request testimonial within 48 hours of final delivery
- Schedule a 90-day follow-up for potential repeat work
