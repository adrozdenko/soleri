# Plan Feature

## When to Use

When architecting a new system, planning a feature before implementation, or designing an approach that spans multiple files or components.

## Steps

### 1. Research

- Search vault for prior art and existing patterns: `op:search_intelligent`
- Search memory for similar past plans: `op:memory_search`
- Read relevant source files to understand current state

### 2. Brainstorm

- Generate at least 2 alternative approaches
- Consider trade-offs: complexity, reversibility, blast radius
- Identify risks and dependencies

### 3. Create Plan

- Create a structured plan: `op:orchestrate_plan`
- Grade must reach B or higher before presenting to user
- Present plan with Plan ID — user must explicitly approve

### 4. Capture

- Capture architectural decisions to vault: `op:capture_knowledge`
- Complete orchestration: `op:orchestrate_complete`
