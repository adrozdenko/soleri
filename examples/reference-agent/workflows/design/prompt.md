# Design System Recommendation

## When to Use

When making visual design decisions: color palettes, typography, UI style, dark/light mode, spacing systems, or component aesthetics.

## Steps

### 1. Research

- Search vault for existing design patterns and decisions: `op:search_intelligent`
- Check for applicable design tokens and semantic color rules

### 2. Recommend

- Provide a full design system recommendation: UI style, palette, typography
- Always use semantic tokens — never raw hex or rgb values
- Check contrast ratios for accessibility (WCAG AA minimum)

### 3. Validate

- Validate token usage against design system rules
- Flag any accessibility violations

### 4. Capture

- Capture design decisions to vault: `op:capture_knowledge`
